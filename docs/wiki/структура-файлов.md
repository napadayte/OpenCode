# Структура файлов

## Глобальный репо (`~/.config/opencode/`)

```
~/.config/opencode/
├── README.md                       вход для людей
├── AGENTS.md                       глобальный контракт для агентов
├── opencode.json                   OpenCode конфиг
├── .opencode/instructions.md       глобальные правила для OpenCode
├── docs/
│   ├── architecture.md             как всё устроено
│   ├── how-to-use.md               расширенный гайд
│   ├── daily-workflow.md           ежедневный цикл
│   ├── agent-shells.md             заметки по каждому шеллу
│   └── wiki/                       👋 ты тут
├── projects/templates/
│   ├── agent-ready-workspace/      шаблон для документации/данных/n8n
│   └── agent-ready-project/        шаблон для кодовых проектов
└── tools/bin/
    ├── new-agent-workspace         создать workspace
    ├── new-agent-project           создать project
    ├── new-agent-doc               добавить агента
    ├── new-skill-doc               добавить скилл
    └── capture-agent-asset         захватить внешний ассет
```

## Шаблон workspace (когда создаёшь новый)

```
my-workspace/
├── 📄 AGENTS.md                    общий контракт
├── 📄 README.md                    вход для людей
├── 📄 CLAUDE.md                    адаптер для Claude
├── 📄 GEMINI.md                    адаптер для Gemini
├── 📄 opencode.json                OpenCode-конфиг
│
├── 📂 .opencode/                   👈 КАНОНИЧЕСКОЕ МЕСТО
│   ├── instructions.md
│   ├── agents/                     роли (планнер, ревьюер, ...)
│   ├── skills/                     процедуры (брейншторм, бэкап, ...)
│   ├── commands/                   слэш-команды (/plan, /review, ...)
│   └── rules/                      правила безопасности и workflow
│
├── 📂 .claude/                     адаптер
│   └── settings.json               запреты на чтение секретов
├── 📂 .codex/                      адаптер
│   └── config.toml
├── 📂 .gemini/                     адаптер
│   └── settings.json
│
├── 📂 docs/                        твои документы
│   ├── overview.md
│   ├── runbook.md
│   ├── plans/                      сюда /plan кладёт планы
│   ├── reports/                    сюда /review кладёт отчёты
│   └── decisions/                  ADR (architecture decision records)
│
├── 📂 data/                        данные
│   ├── inventory.md                реестр данных
│   ├── schemas/
│   ├── exports/                    (gitignored)
│   └── backups/                    (gitignored)
│
├── 📂 automations/                 n8n и автоматизации
├── 📂 research/                    исследования
├── 📂 notes/                       заметки

├── 📂 checklists/                  чек-листы
└── 📂 tools/                       скрипты, сканеры
```

## Главное правило

> [!important]
> **Канонические файлы — в `.opencode/`.**
> Остальные адаптеры (`.claude/`, `.codex/`, `.gemini/`) — это только настройки конкретного шелла, не дублируют контент.

## Минимальный набор для нового проекта

Если делаешь не workspace, а простой проект — достаточно этих файлов:

```
my-project/
├── AGENTS.md            общий контракт
├── README.md            для людей
├── docs/architecture.md
└── docs/runbook.md
```
