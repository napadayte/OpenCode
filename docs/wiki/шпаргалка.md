# Шпаргалка

Всё, что нужно знать на одной странице.

## Создание

```bash
new-agent-workspace <name>     # docs / data / n8n / research / scripts
new-agent-project <name>       # код-первый проект
new-agent-doc <slug>           # добавить агента (внутри workspace)
new-skill-doc <slug>           # добавить скилл (внутри workspace)
capture-agent-asset <file>     # затащить внешний материал
```

## Шеллы — запуск

```bash
opencode      # основной
claude        # Claude Code
codex         # Codex CLI
gemini        # Gemini CLI
```

## Команды внутри OpenCode

```
/analyze          ориентировка (read-only)
/plan <task>      план в docs/plans/
/review           проверка диффа (read-only)
/intake <path>    разобрать внешний материал
/inventory <a>    обновить data/inventory.md
```

## Дневной цикл

```
git status                           # до
opencode
  /analyze                           # ориентировка
  /plan <task>                       # план
  (реализация вручную из плана)
  /review                            # проверка
git status                           # после
git diff
git commit -m "..."                  # вручную
```

## 4 концепции

| | Где | Пример |
|---|---|---|
| Агент | `.opencode/agents/<slug>.md` | `planner`, `reviewer` |
| Скилл | `.opencode/skills/<slug>/SKILL.md` | `brainstorm`, `backup-restore` |
| Правило | `.opencode/rules/<slug>.md` | `security.md`, `git-operations.md` |
| Команда | `.opencode/commands/<slug>.md` | `/plan`, `/review` |

## Безопасность — основное

| Действие | Правило |
|---|---|
| Push | Только вручную |
| Commit | Только вручную |
| `rm -rf` | Спросить |
| Секреты в коде | Никогда |
| `git push --force` | Никогда без явного "да" |
| Docker volume rm | Никогда без бэкапа |

## Минимальный безопасный шаблон permission

```json
{
  "permission": {
    "*": "ask",
    "read": "allow",
    "edit": "ask",
    "bash": {
      "*": "ask",
      "git status*": "allow",
      "git diff*": "allow",
      "git push*": "deny",
      "rm -rf *": "deny"
    }
  }
}
```

## Минимальный AGENTS.md (стартовый)

```markdown
# AGENTS.md

## Rules
- Do not push.
- Do not commit unless explicitly requested.
- Ask before destructive commands.
- Protect secrets.
- Stop and re-plan on surprises.

## First action
1. Classify task.
2. Read relevant rules.
3. Plan before editing.
4. Report changed files + verification + risks.
```

## Чего больше нет (выпилено)

- ❌ `/fix` команда — реализация теперь **вручную из плана**
- ❌ `name:` в agent frontmatter — OpenCode берёт имя из имени файла
- ❌ `model: sonnet` (bare alias) — используй `model: anthropic/claude-sonnet-4-6` или убери поле
- ❌ `tools: [Read, Edit]` (массив) — это Claude Code формат; у OpenCode разрешения через `permission:`
- ❌ `allowed-tools:`, `risk:`, `source:` в skill frontmatter
- ❌ Inline `agent:` / `command:` блоки в workspace `opencode.json`
