# Глобальные агенты

> 27 специалистов, доступных во **всех** workspace. Подключены через `~/.config/opencode/agents/` (в репо — `<repo>/agents/`).

## Откуда

Адаптировано из [VoltAgent/awesome-claude-code-subagents](https://github.com/VoltAgent/awesome-claude-code-subagents) под спеку OpenCode:

- Дропнули `name:` (имя берётся из имени файла)
- Дропнули `tools:` CSV (был формат Claude Code)
- Дропнули `model:` (агенты модель-агностичны — наследуют `default_agent`)
- Добавили `mode: subagent` (не светятся в `/agent` пикере — вызываются через `manager` или явный `task`)
- Добавили `permission:` блок (safe defaults: `edit: ask`, deny на push/rm-rf/sudo)
- Добавили рекомендацию по тиру модели в тело (haiku/sonnet/opus → таблица для Anthropic/OpenAI/Gemini)

## Как вызвать

Все агенты — `mode: subagent`, значит они **не появляются** в `Tab` пикере. Два способа:

### 1. Через manager (рекомендуется)

```
> manager: поищи мне трендов про AI agents за 2026
```
`manager` сам поймёт что нужен `trend-analyst` и вызовет его через `task` tool.

### 2. Явно через task

В любом primary агенте:
```
> используй task с агентом research-analyst чтобы собрать...
```

## Модель — что выбирать

В frontmatter `model:` нет — каждый агент наследует `default_agent`. Но в **теле** агента есть таблица рекомендаций по тиру (haiku/sonnet/opus) с конкретными model id для Anthropic/OpenAI/Gemini.

Если хочешь жёстко привязать агента к модели — добавь `model: anthropic/claude-haiku-4-5` в его frontmatter.

## Permission (для всех 27)

```yaml
permission:
  edit: ask
  bash:
    "*": ask
    pwd|ls|cat|rg|grep|find: allow
    git status|diff|log: allow
    git push|rm -rf|sudo: deny
```

Если конкретному агенту нужны другие права — отредактируй его файл напрямую.
