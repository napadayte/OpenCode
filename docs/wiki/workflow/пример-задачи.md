# Пример задачи

Полный цикл — настоящая задача от начала до коммита.

## Задача

> Добавить еженедельный n8n воркфлоу, который экспортирует список клиентов в Google Sheets.

## Этап 1 — ориентировка

```bash
cd ~/code/my-workspace
git status                # чисто
opencode
```

```
/analyze
```

Агент возвращает: workspace для документации, есть `data/inventory.md`, `automations/n8n/`. Никаких клиентов пока в инвентаре нет.

## Этап 2 — план

```
/plan Добавить n8n воркфлоу: weekly export of customers → Google Sheets
```

Агент видит:
- задача = automation + data
- читает `automation-operations.md`, `data-operations.md`, `security.md`
- задаёт **один** уточняющий вопрос:

> Откуда брать клиентов — Supabase, локальная SQLite, или manual upload? И есть ли уже Google credentials в n8n?

Ответ:

```
Из локальной SQLite ./data/customers.sqlite.
Google credentials уже есть, имя — "google-sheets-prod".
```

Агент создаёт `docs/plans/n8n-customers-export.md`:

```markdown
# n8n: Weekly customers export → Google Sheets

## Goal
Раз в неделю выгружать customers.sqlite → Google Sheet "Customers Weekly"

## Affected files
- automations/n8n/customers-export-weekly.md  (НОВЫЙ — документация workflow)
- data/inventory.md  (обновить — добавить Google Sheet endpoint)

## Affected data
- data/customers.sqlite  (read-only)
- Google Sheet "Customers Weekly"  (write)

## Steps
1. Создать n8n workflow:
   - Trigger: Cron Mon 08:00
   - Read SQLite node → SELECT * FROM customers
   - Format columns
   - Google Sheets node → Append/Update
2. Документировать в automations/n8n/customers-export-weekly.md
3. Обновить data/inventory.md (sheet — это новый external data asset)
4. Проверить что credentials google-sheets-prod не закоммичены

## Risks
- Если SQLite заблокирован — workflow упадёт (low)
- Если credentials в Git → leak (CRITICAL — проверить .gitignore)
- Sheet с PII попадёт в общий Drive (medium — настроить sharing manually)

## Verification
- Прогон workflow вручную (n8n Execute)
- Проверить что Sheet заполнился
- git status — никаких .json с credentials

## Rollback
- Disable workflow в n8n (не удалять — оставить для recovery)
- Document отключение в runbook

## Approvals needed
- Подтверждение, что Sheet нужен (sharing settings)
```

## Этап 3 — реализация (вручную)

Ты читаешь план. Идёшь в n8n UI, делаешь воркфлоу. Возвращаешься в OpenCode и просишь:

```
> Сделай automations/n8n/customers-export-weekly.md по плану из docs/plans/n8n-customers-export.md
> Сделай через @automation-operator agent (n8n-workflow-doc skill)
```

Агент создаёт документацию.

Ты вручную обновляешь `data/inventory.md` или просишь:

```
/inventory ./data/customers.sqlite
```

## Этап 4 — ревью

```
/review
```

Reviewer проходит по диффу. Security-reviewer проверяет, что credentials нигде не светятся. Возвращает:

```
APPROVE

Critical: none
Important:
  - automations/n8n/customers-export-weekly.md:42
    Failure mode не указывает что делать если SQLite locked.
Suggestions:
  - data/inventory.md:67 — last_verified пустое, рекомендую заполнить
    после первого run.
```

## Этап 5 — фикс важного

Открываешь файл, исправляешь Failure mode секцию.

```
/review
```

```
APPROVE — no remaining issues.
```

## Этап 6 — коммит

```bash
# выходишь из opencode (Ctrl+D или /exit)

git status
# modified:   data/inventory.md
# new file:   automations/n8n/customers-export-weekly.md
# new file:   docs/plans/n8n-customers-export.md

git diff
# проверяешь что нет .json с credentials, нет sqlite в diff

git add automations/n8n/customers-export-weekly.md \
        data/inventory.md \
        docs/plans/n8n-customers-export.md
git commit -m "Add weekly customers export n8n workflow (sqlite → Google Sheets)"

# git push — ВРУЧНУЮ, и только если действительно нужно публиковать
```
