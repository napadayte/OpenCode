# Workflow

Как работать каждый день.

## Главный цикл

```mermaid
flowchart LR
    G1[git status<br/>до]
    G1 --> S[opencode]
    S --> A[/analyze]
    A --> P[/plan task]
    P --> M[Реализация<br/>вручную]
    M --> R[/review]
    R --> G2[git status<br/>после]
    G2 --> C[Manual commit]
```

## Когда что использовать

| Этап | Команда / действие |
|---|---|
| **Открыл проект впервые** | [[первый-промпт]] или `/analyze` |
| **Понял задачу** | `/plan <task>` |
| **Реализуешь** | вручную, читая план |
| **Перед коммитом** | `/review` |
| **Коммит** | вручную, после ревью |

## Странички

- [[первый-промпт]] — готовый текст для копи-пейста
- [[дневной-цикл]] — пошаговый цикл с командами
- [[пример-задачи]] — реальный пример от и до
- [[взгляд-агента]] — как агент думает изнутри

## Принципы

> [!important]
> 1. **Classify first** — что за задача (план, фикс, бэкап)?
> 2. **Plan before changes** — план в `docs/plans/` если >3 файлов
> 3. **Manual commit** — агент не коммитит сам
> 4. **Stop on surprise** — неожиданный diff = стоп
