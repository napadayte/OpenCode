# Первый промпт

Когда открываешь новый workspace — скопируй и вставь:

```
Analyze this workspace. Do not edit files.
Read AGENTS.md, README.md, docs/overview.md, docs/runbook.md,
.opencode/rules/dispatch-policy.md, and .opencode/rules/workflow.md.
Summarize structure, risks, and next steps.
```

Или просто:

```
/analyze
```

## Что произойдёт

```mermaid
sequenceDiagram
    участник Ты
    участник Агент
    Ты->>Агент: /analyze
    Агент->>Агент: Читает AGENTS.md
    Агент->>Агент: Читает rules/
    Агент->>Агент: Смотрит структуру
    Агент-->>Ты: Резюме: workspace, риски, шаги
```

## Что получишь в ответ

1. **Workspace purpose** — что это за проект
2. **Active workstreams** — что сейчас в работе
3. **Structure** — основные папки
4. **Risks** — где может сломаться
5. **Next recommended step** — куда идти дальше

## Зачем это вообще

Без `/analyze` агент не знает:
- какие правила в проекте
- какие данные есть
- что нельзя коммитить
- что уже сделано

С `/analyze` агент **прочитал контракт** и готов работать безопасно.
