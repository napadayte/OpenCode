# AGENTS.md

Shared contract for every AI shell that opens this repo (OpenCode, Claude Code, Codex CLI, Gemini CLI).

This file is the **single source of truth across shells**. OpenCode-only rules live in `.opencode/instructions.md` and are loaded automatically via `opencode.json`.

## Repo purpose

Global config workspace for AI coding shells. Contains:

- `opencode.json` — OpenCode runtime config (top-level + per-agent permissions, commands).
- `agents/` — global OpenCode agents (subagents available across projects).
- `skills/` — global OpenCode skills.
- `projects/templates/` — generator templates (`agent-ready-project`, `agent-ready-workspace`).
- `tools/bin/` — bash generators (linked into `~/.local/bin` by `install-opencode-tools`).
- `docs/` — architecture, per-shell notes, Russian wiki, audits.

See `docs/architecture.md` for the full tree and workflow.

## Rules — apply in every shell

1. Follow existing architecture and style. Small, reviewable changes.
2. Ask before adding dependencies.
3. **Do not push.** **Do not commit unless explicitly requested.**
4. **Ask before destructive commands** (`rm -rf`, `git reset --hard`, `git push --force`, `docker volume rm`, format/drop/migrate).
5. Never commit secrets, API keys, auth files, tokens, database dumps, or local backups (see `.gitignore`).
6. Keep templates generic — no machine-specific paths, no credentials.
7. After any change, summarize: changed files, why, how to verify, remaining risks, next step.

## OpenCode-specific runtime

`opencode.json` defines three primary/subagent roles for working **on this config repo itself**:

- `architect` — read-only planning.
- `implementer` — edits with `ask` permission.
- `reviewer` — read-only review (subagent).

Workspaces generated by `new-agent-workspace` use a different agent set centered on a `manager` orchestrator — see `projects/templates/agent-ready-workspace/AGENTS.md`. Do not confuse the two: this file governs the **config repo**; the template governs **generated workspaces**.

## Per-shell notes

- Claude Code: `.claude/settings.json` (deny rules) and `docs/shells/claude/`.
- Codex CLI: `docs/shells/codex/AGENTS.md`.
- Gemini CLI: `docs/shells/gemini/`.
- OpenCode: `.opencode/instructions.md` and `docs/shells/opencode/`.

The templates under `projects/templates/` ship per-shell entry files (`CLAUDE.md`, `GEMINI.md`) that all `@AGENTS.md` — only generated projects need those mirrors.
