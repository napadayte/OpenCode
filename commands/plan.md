description: Produce a step-by-step plan before changing code
---

Create a step-by-step implementation plan for: $ARGUMENTS

Do not edit files yet.

Required sections:
- **Goal** — what success looks like, in one sentence.
- **Affected files** — concrete paths, not vague areas.
- **Steps** — ordered, each step small enough to review in isolation.
- **Risks** — what could break (other features, tests, data, security).
- **Test plan** — how to verify each step + the whole change.
- **Rollback plan** — how to undo if something goes wrong (git revert, manual cleanup, etc.).
- **Out of scope** — what this plan deliberately does NOT change.

If the task is unclear, list the open questions before drafting steps. Do not guess.
