# Data Inventory Skill

------------------------------------------------------------

## Purpose

Document data assets, paths, sensitivity, git policy, backup policy, and restore notes.

------------------------------------------------------------

## When To Use

Use this skill when the task matches this procedure and may be repeated later.

------------------------------------------------------------

## Inputs Needed

Identify:

- goal
- environment
- affected files
- affected data
- risks
- expected output
- approval needed

------------------------------------------------------------

## Steps

1. Classify the task.
2. Read relevant rules.
3. Identify affected files and data.
4. Identify risks.
5. Create or follow a short plan.
6. Perform only approved actions.
7. Verify the result.
8. Report changed files, verification, risks, and next step.

------------------------------------------------------------

## Output Format

Return:

- summary
- steps performed or proposed
- affected files
- affected data
- risks
- verification
- next step

------------------------------------------------------------

## Safety Rules

- do not expose secrets
- do not delete data without approval
- do not commit unless requested
- do not push unless requested
- stop and ask when uncertain

------------------------------------------------------------
